# Git Notes 📘

A structured, from-scratch-to-advanced set of notes for learning **Git** and **GitHub** — covering core concepts, everyday commands, branching, remotes, history recovery, advanced tools, GitHub workflows, best practices, and interview preparation.

## 🗺️ Learning Roadmap

1. **01-Introduction** — What Git is, why it exists, how it differs from GitHub, and its internal architecture.
2. **02-Installation-and-Setup** — Installing Git, configuring your identity, setting up SSH keys, and connecting to GitHub.
3. **03-Basic-Commands** — The everyday commands: init, clone, status, add, commit, log, diff, rm, mv, and ignoring files.
4. **04-Branching** — Creating and switching branches, merging, and resolving conflicts.
5. **05-Remote-Repositories** — Working with remotes: fetch, pull, push, and upstream tracking.
6. **06-History-and-Recovery** — Undoing mistakes safely: reset, revert, restore, reflog, and stash.
7. **07-Advanced-Git** — Rebase, cherry-pick, tags, bisect, blame, clean, and submodules.
8. **08-GitHub-Workflow** — Forking, pull requests, code review, and branching strategies (GitFlow, GitHub Flow, Trunk-Based).
9. **09-Best-Practices** — Writing good commit messages, naming branches, structuring repos, and avoiding common mistakes.
10. **10-Interview** — Interview Q&A, scenario-based questions, and a quick cheat sheet.
11. **Resources** — Useful links, a command cheat sheet, and practice exercises.

## 📌 How to Use These Notes

- Follow the folders in numerical order if you're learning Git from scratch.
- Jump directly to a topic (e.g., `07-Advanced-Git/git-rebase.md`) if you already know the basics.
- Use `10-Interview/` before interviews for a quick refresher.
- Use `Resources/Commands-Cheat-Sheet.md` as a fast lookup while working.

## ✅ Prerequisites

- Basic command-line familiarity (navigating folders, running commands).
- No prior Git knowledge required — Section 01 starts from zero.

Happy learning! 🚀
